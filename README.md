# Skills

这个仓库用于维护个人通用 skills。

## Skills 列表

### grill-me

用于通过连续、聚焦的追问来压力测试计划或设计，直到关键假设、取舍和决策都足够清晰。

### karpathy-guidelines

用于编码、代码评审或重构时约束行为，帮助保持修改简单、聚焦且可验证。

## 目录结构

每个 skill 使用独立目录，并包含一个 `SKILL.md` 文件：

```text
<skill-name>/
  SKILL.md
```

## 维护说明

- 每个 skill 应聚焦在单一行为或工作流上。
- `SKILL.md` 中应优先写清楚触发条件和适用边界。
- 只有当 skill 确实需要时，才添加辅助脚本、模板或资源文件。
